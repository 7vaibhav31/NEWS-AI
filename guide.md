# 🚀 NEWS-AI Setup & Developer Guide

Welcome to the **NEWS-AI** project! This guide is designed to help you set up, run, and develop on this codebase as quickly and easily as possible.

---

## 📂 Project Structure

This repository is split into two main sections:
1. **`news/`**: The backend admin panel built with **Laravel**. It contains the database models, migrations, AI integrations (Gemini, Claude, OpenAI, Groq), and configuration screens.
2. **`newsnow/`**: The frontend website. This is a pre-built static export of the **Next.js** frontend application.

---

## 🛠️ Prerequisites

Before you start, make sure you have the following installed on your machine:
* **PHP (8.1 or higher)**
* **Composer** (PHP Package Manager)
* **MySQL/MariaDB** (Usually via **XAMPP** or **Laragon**)
* A text editor (e.g., VS Code or Cursor)

---

## 🏁 Setup Options

You can set up and run the code in one of two ways:
* **Option A: Local Setup** (Uses XAMPP/Laragon on your physical machine)
* **Option B: Cloud Setup** (Uses GitHub Codespaces online VS Code; **no local installs required**)

---

## 💻 Option A: Local Setup (XAMPP / Laragon)

### Step 1: Database Setup
1. Open your XAMPP or Laragon Control Panel and start **Apache** and **MySQL**.
2. Open **phpMyAdmin** in your browser: `http://localhost/phpmyadmin`
3. Create a brand new database named `u948859228_news` (Set collation to `utf8mb4_unicode_ci` if prompted).
4. Click on your new database, navigate to the **Import** tab.
5. Choose the database dump file **`u948859228_news.sql`** located in the root of this project folder, and click **Import/Go**.

> [!TIP]
> **Windows/MariaDB Connection Issue Troubleshooting**
> If Laravel fails to connect to MySQL (e.g., showing `SQLSTATE[HY000] [2002]`), this is usually because MariaDB on Windows binds only to IPv6 by default.
> * Open your XAMPP configuration folder (usually `C:\xampp\mysql\bin\my.ini`).
> * Search for `[mysqld]` and add the following line right below it:
>   ```ini
>   bind-address=0.0.0.0
>   ```
> * Restart MySQL in your XAMPP control panel.

### Step 2: Backend Setup (Laravel)
1. Open a terminal and navigate to the `news/` directory:
   ```bash
   cd news
   ```
2. Copy `.env.example` to create your own configuration file:
   ```bash
   copy .env.example .env
   ```
   *(Or just edit the pre-existing `.env` file).*
3. Open the `.env` file in your editor and verify your database details match XAMPP defaults:
   ```env
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=u948859228_news
   DB_USERNAME=root
   DB_PASSWORD=
   ```
4. Install all PHP dependencies:
   ```bash
   composer install
   ```
5. Run the database migrations. This will add the latest AI News columns and Settings tables to your database:
   ```bash
   php artisan migrate
   ```
6. Link the public storage folder so images generated by the AI are visible in the browser:
   ```bash
   php artisan storage:link
   ```
7. Start the backend local server:
   ```bash
   php artisan serve
   ```
   🎉 Your Laravel backend is now running at **`http://127.0.0.1:8000`**!

### Step 3: Login & Admin Settings
1. Go to the Admin Panel: `http://127.0.0.1:8000/admin` (or `http://127.0.0.1:8000/` which redirects to the login screen).
2. Log in using the default developer credentials:
   * **Username:** `admin` (or Email: `admin@gmail.com`)
   * **Password:** `admin123`
3. Navigate to **News Management** -> **AI News Engine** in the sidebar.
4. Set up your AI providers:
   * **Text Generator Providers**: You can choose between **Google Gemini**, **OpenAI**, **Claude**, or **Groq**.
   * **API Keys**: Insert your API keys. They are securely encrypted before being saved to the database.
   * **Image Generator Provider**: 
     * Select **Free AI (Pollinations.ai)** to generate cover images instantly *without* needing any API keys.
     * Select **OpenAI (DALL-E)** to use your own OpenAI key for high-quality images.
5. Click **Save Settings**.
6. Try a test generation by clicking the green **Generate Now (test run)** button! You can inspect the results under the **View Logs** page.

### Step 4: Frontend Setup (Next.js Static Export)
1. Open a new terminal window/tab and navigate to the `newsnow/` directory from the root:
   ```bash
   cd newsnow
   ```
2. Since this folder contains a static export, you can host it using any simple local server. For example, using PHP's built-in server:
   ```bash
   php -S 127.0.0.1:3000
   ```
3. Open your browser and go to **`http://127.0.0.1:3000`** to view the frontend!

## ☁️ Option B: Zero-Install Cloud Setup (GitHub Codespaces)

GitHub Codespaces spins up a secure development container in the cloud. Your friend does not need to install PHP, Composer, or MySQL on their local computer!

### Step 1: Open the Repository in Codespaces
1. Go to the repository on GitHub.
2. Click the green **Code** button (top-right of the repo home page).
3. Select the **Codespaces** tab and click **Create codespace on main**.
4. The cloud environment will start up, launching a browser-based version of VS Code.

### Step 2: Initialize MySQL inside the Codespace
Codespace containers run on Linux and come with MySQL preinstalled but stopped by default. 
1. Look at the bottom pane to find the **Terminal** (If not visible, press `Ctrl + \`` or `Cmd + \`` to open it).
2. Start the MySQL service:
   ```bash
   sudo service mysql start
   ```
3. Create the database:
   ```bash
   sudo mysql -u root -e "CREATE DATABASE u948859228_news CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
   ```
4. Import the database dump file `u948859228_news.sql` from the project root:
   ```bash
   sudo mysql -u root u948859228_news < u948859228_news.sql
   ```

### Step 3: Configure Environment & Install Dependencies
1. Navigate to the `news/` directory in your terminal:
   ```bash
   cd news
   ```
2. Copy the example configuration to `.env`:
   ```bash
   cp .env.example .env
   ```
3. Open the `.env` file in the online VS Code explorer and verify/update these database values (they should align with the Codespace default MySQL setup):
   ```env
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=u948859228_news
   DB_USERNAME=root
   DB_PASSWORD=
   ```
4. Install PHP dependencies inside the container:
   ```bash
   composer install
   ```
5. Run the migrations to apply the database schema changes:
   ```bash
   php artisan migrate
   ```
6. Link the public storage:
   ```bash
   php artisan storage:link
   ```

### Step 4: Run the Backend & Frontend Servers
1. Start the Laravel backend:
   ```bash
   php artisan serve
   ```
2. **Accessing the Admin Panel**: 
   * A popup will appear in the bottom-right corner of the online VS Code stating: *"Your application running on port 8000 is available."*
   * Click **Open in Browser**. The URL will look similar to: `https://your-codespace-name-8000.app.github.dev/admin`
   * Log in using:
     * **Username/Email**: `admin@gmail.com`
     * **Password**: `admin123`
3. **Running the Frontend (Optional)**:
   * Open a new terminal tab in Codespaces (click the `+` icon on the terminal header).
   * Navigate to `newsnow/`:
     ```bash
     cd newsnow
     ```
   * Start the static web server:
     ```bash
     php -S 127.0.0.1:3000
     ```
   * A popup will appear for port `3000`. Click **Open in Browser** to view the frontend site at `https://your-codespace-name-3000.app.github.dev/`!

## 💡 Code Architecture Quick Reference

Here are the key files you should know about if you want to modify or add features:

* **AI Providers & Integrations**: Located in [`news/app/Services/AiNews/Providers/`](file:///d:/intern_project/news/app/Services/AiNews/Providers/)
  * [GeminiProvider.php](file:///d:/intern_project/news/app/Services/AiNews/Providers/GeminiProvider.php) - Handles prompt sending to Google Gemini.
  * [GroqProvider.php](file:///d:/intern_project/news/app/Services/AiNews/Providers/GroqProvider.php) - Handles prompt sending to Groq.
  * [ClaudeProvider.php](file:///d:/intern_project/news/app/Services/AiNews/Providers/ClaudeProvider.php) - Handles prompt sending to Anthropic Claude.
  * [OpenAiProvider.php](file:///d:/intern_project/news/app/Services/AiNews/Providers/OpenAiProvider.php) - Handles prompt sending to OpenAI.
* **Core Logic Service**: [AiNewsService.php](file:///d:/intern_project/news/app/Services/AiNews/AiNewsService.php)
  * This is where prompts are constructed, articles are parsed, categories are mapped, and the AI image generator is called.
* **Database Migrations**: [database/migrations/](file:///d:/intern_project/news/database/migrations/)
  * Contains the database tables configuration.
* **Routes**: Configured in `news/routes/web.php`. Look for `AiNewsController` routes.
* **Cron Command**: [GenerateAiNews.php](file:///d:/intern_project/news/app/Console/Commands/GenerateAiNews.php)
  * Can be triggered manually via terminal using:
    ```bash
    php artisan ai-news:generate
    ```

## 🛠️ Helpful Terminal Commands

* **Clear Laravel cache (run this if settings aren't updating)**:
  ```bash
  php artisan optimize:clear
  ```
* **Run migrations manually**:
  ```bash
  php artisan migrate
  ```
* **Roll back migrations**:
  ```bash
  php artisan migrate:rollback
  ```
